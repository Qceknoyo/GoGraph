# GoGraph! 📊

Веб-сервис для загрузки, обработки и построения графиков по результатам испытаний материалов. .

  <img src="assets/demo.gif" alt="Demo" width="700">
</p>

## Что умеет проект

- Загрузка данных из `.txt` и `.csv` файлов.
- Парсинг деформации/напряжения и расчет точек разрушения.
- Интерактивный график на фронтенде (Chart.js).
- Сохранение истории тестов в БД с возможностью полной очистки.
- Автоматические сессии по кукам (работает без регистрации).
- Синхронизация сессии на другом устройстве по одноразовой ссылке.

## Стек

- **Бэкенд:** Go (пакеты `net/http`, `database/sql`, драйвер `lib/pq`).
- **База данных:** PostgreSQL 16.
- **Фронтенд:** Чистый JS (SPA-логика), HTML5, CSS3.
- **Контейнеры:** Docker, Docker Compose.

## Запуск

Проект полностью упакован в Docker, поэтому настраивать локально базу и Go не нужно. 

1. Сделайте файл `.env` в корне и пропиши доступы (как в `.env.example`).
2. Подними контейнеры командой:
```bash
sudo docker-compose up --build
```

Сервер поднимется здесь: **[http://localhost:8181](http://localhost:8181)**  
Порт базы данных прокинут наружу на: `localhost:5433`

## Структура файлов

```text
.
├── main.go            # Запуск сервера, подключение к БД и роутинг
├── session.go         # Middleware сессий, чистка старых токенов и синхронизация
├── Dockerfile         # Инструкция для сборки легкого Docker-образа (multi-stage)
├── docker-compose.yml # Манифест для одновременного запуска сервера и Postgres
├── .env               # Конфиг с паролями базы (игнорируется гитом)
├── .gitignore         # Список файлов, которые не летят в репозиторий
├── db/
│   └── init.sql       # Схема таблиц (накатывается сама при первом старте)
├── templates/
│   └── spa.html       # Главный HTML-шаблон приложения
└── ui/
    └── static/        # Скрипты и стили фронтенда
        └── uploads/   # Папка для загружаемых файлов
```
